```python
from opentelemetry import metrics
from opentelemetry.sdk.metrics import MeterProvider
from opentelemetry.sdk.metrics.export import InMemoryMetricReader

# Setup
reader = InMemoryMetricReader()
meter_provider = MeterProvider(metric_readers=[reader])
metrics.set_meter_provider(meter_provider)
meter = metrics.get_meter(__name__)

# Create instruments
counter = meter.create_counter("requests.total", unit="1", description="Total requests processed")
histogram = meter.create_histogram("request.duration", unit="ms", description="Request processing time")

# Record observations
counter.add(1, {"endpoint": "/checkout", "status": "ok"})
histogram.record(142, {"endpoint": "/checkout"})
```

Phase 1 description (concept-first, Markdown):
1. "What are Metrics in OpenTelemetry?" — 2-3 sentences. Metrics are numeric measurements aggregated over time — counters, gauges, and histograms. Unlike spans (which capture individual request details), metrics give you statistical views: p99 latency, error rate, throughput.
2. "Metrics vs. Traces — Two Different Questions" — clear distinction: metrics answer "what is the overall health?", traces answer "why did this specific request fail?". You need both: metrics alert you, traces explain the cause.
3. "The OTel Metrics API" — brief: MeterProvider (like TracerProvider), Meter (like Tracer), Counter (monotonically increasing), Histogram (distribution of values), Gauge (point-in-time value).
4. "Why it matters in production" — anecdote: "Our p99 latency was 1500ms but individual traces looked fine at 200ms. Only by correlating the histogram with untraced code segments did we find a serialization step adding 1.3s outside the traced path."
5. "The Situation:" — a checkout API is showing p99 latency of 1500ms in the metrics dashboard, but engineers examining individual traces see only 200ms handler spans. The gap is unexplained.
6. "Your Mission:" — Create a MeterProvider and Meter. Inside the traced checkout handler, record a `requests.total` counter and a `request.duration` histogram. Add a span attribute `metrics.recorded: 'true'` to confirm your metric code ran.
