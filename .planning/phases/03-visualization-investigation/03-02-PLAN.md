Based on 03-RESEARCH.md:
1. Filter State Loss on Tab Switch: If LogViewer unmounts on tab switch, filter state is lost. Use display:none instead of conditional rendering.
2. Log Timestamp Parsing: Use string-based timestamps (HH:MM:SS.mmm) — simpler for display, matches Phase2Data format.
3. Log Generation Granularity: Don't generate too many logs (overwhelming) or too few (not educational). 2-4 per span is the sweet spot.
</common_pitfalls>

  <name>Task 1: Create synthetic log generation library</name>
  <files>src/lib/logGenerator.ts</files>
  <action>
    Create logGenerator.ts that generates realistic log entries from span data:
    
    1. **Define LogEvent interface:**
       ```typescript
       interface LogEvent {
         name: string;
         level: 'info' | 'warn' | 'error';
         message: string;
         timestamp: number;  // Relative ms from trace start
       }
       ```
    
    2. **createSpanStartLog(span):** Generate info log when span starts
       - Level: 'info'
       - Message: `Starting ${span.name}`
       - Timestamp: span.offsetMs + 1 (slight offset to avoid collision)
       
    3. **createSpanEndLog(span):** Generate info log when span ends
       - Level: 'info'
       - Message: `Completed ${span.name} in ${span.durationMs}ms`
       - Timestamp: span.offsetMs + span.durationMs - 1 (just before end)
       
    4. **createWarningLog(span):** Generate warning log for slow spans
       - Only if span.status === 'warning'
       - Check span.attributes for contextual hints:
         - 'db.connection_pool.wait_ms' → `DB connection pool exhausted, waiting ${waitMs}ms`
         - 'http.request.duration' > 100 → `Slow HTTP request detected (${duration}ms)`
         - 'cache.miss' === 'true' → `Cache miss for key ${cacheKey}`
       - Timestamp: span.offsetMs + 10 (shortly after start)
       
    5. **createErrorLog(span):** Generate error log for error spans
       - Only if span.status === 'error'
       - Check span.attributes:
         - 'error.message' → Use this message
         - 'error.type' → Include type in message
         - 'http.status_code' → `HTTP error ${statusCode}`
       - Timestamp: span.offsetMs + span.durationMs - 5 (near end)
       
    6. **generateLogsFromSpans(spans, traceId, traceStartMs):** Main function
       - Input: TraceSpan[] (already transformed), traceId, trace start time
       - For each span, generate:
         - Start log (always)
         - Warning log (if slow and has relevant attributes)
         - Error log (if error status)
         - End log (always)
       - Sort all logs by timestamp
       - Convert timestamps to HH:MM:SS.mmm format using traceStartMs
       - Return LogEntry[]
       
    7. **Helper: formatTimestamp(relativeMs, traceStartMs):**
       ```typescript
       function formatTimestamp(relativeMs: number, traceStartMs: number): string {
         const totalMs = traceStartMs + relativeMs;
         const date = new Date(totalMs);
         const hh = date.getHours().toString().padStart(2, '0');
         const mm = date.getMinutes().toString().padStart(2, '0');
         const ss = date.getSeconds().toString().padStart(2, '0');
         const mmm = Math.floor(date.getMilliseconds()).toString().padStart(3, '0');
         return `${hh}:${mm}:${ss}.${mmm}`;
       }
       ```
       
    8. Handle edge cases:
       - Empty spans array → return empty array
       - Missing traceStartMs → use current time as base
       - Malformed attributes → skip that log, don't crash
    
    Export all functions with proper TypeScript types.
