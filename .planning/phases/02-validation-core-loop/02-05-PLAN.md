<task type="auto">
  <name>Prevent auto-save from overwriting on initial mount</name>
  <files>src/App.tsx</files>
  <action>
    Add an initialLoadRef to prevent the auto-save effect from overwriting persisted data before it has been loaded. This creates a guard that prevents saving until after the initial load is complete.
    
    Steps:
    1. Import useRef from React (already imported as useState, useEffect - add useRef)
    2. Add a ref to track initial load: `const initialLoadRef = useRef(true);`
    3. Modify the auto-save useEffect (lines 71-75) to skip saving on initial mount:
    
    ```typescript
    // Code auto-save effect
    useEffect(() => {
      if (isLoaded && !initialLoadRef.current) {
        saveCode(currentCaseId, code);
      }
      initialLoadRef.current = false;
    }, [code, currentCaseId, isLoaded, saveCode]);
    ```
    
    This prevents the race condition where:
    1. Code is initialized with default value
    2. Auto-save triggers and overwrites persisted data
    3. Load effect tries to restore but data is already gone
    
    Now the flow is:
    1. Code initialized with default value
    2. Initial load effect restores persisted code (if exists)
    3. Auto-save starts working after initial load
  </action>
