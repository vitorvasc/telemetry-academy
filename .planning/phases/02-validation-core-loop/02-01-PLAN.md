1. Define core types per research:
   ```typescript
   export type ValidationCheckType = 
     | 'span_exists' 
     | 'attribute_exists' 
     | 'attribute_value'
     | 'span_count'
     | 'status_ok'
     | 'status_error';
   
   export interface SpanValidationRule {
     type: ValidationCheckType;
     spanName?: string;
     attributeKey?: string;
     attributeValue?: any;
     minCount?: number;
     description: string;
     successMessage: string;
     errorMessage: string;
     hintMessage?: string;
     guidedMessage?: string;
   }
   
   export interface ValidationContext {
     spans: any[];
     attemptHistory: Record<string, number>; // rule description -> attempt count
   }
   
   export interface ValidationResult extends SpanValidationRule {
     passed: boolean;
     message: string;
     attemptsOnThisRule: number;
   }
   ```

2. Implement `validateSpans()` function:
   - Takes rules array and ValidationContext
   - Maps over rules, calling appropriate check function for each type
   - Uses attemptHistory to select appropriate message (error -> hint -> guided)
   - Returns ValidationResult[] with passed status and selected message

3. Implement check helpers per OpenTelemetry span structure:
   - `checkSpanExists(spans, spanName)` - exact name match
   - `checkAttributeExists(spans, spanName?, attributeKey)` - checks span.attributes object
   - `checkAttributeValue(spans, spanName?, attributeKey, attributeValue)` - exact value match
   - `checkStatus(spans, spanName?, statusCode)` - checks status.status_code field
   - `checkSpanCount(spans, minCount)` - validates minimum span count

4. Implement progressive message selection:
   - `selectMessage(rule, attempts)` returns:
     - attempts === 0: errorMessage
     - attempts 1-2: hintMessage || errorMessage  
     - attempts >= 3: guidedMessage || hintMessage || errorMessage

5. Export all public types and functions.

Use native TypeScript pattern matching. Do NOT add Zod or other validation libraries per RESEARCH.md decision.
