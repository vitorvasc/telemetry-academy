<objective>
Fix react-resizable-panels v4 Panel size props — all bare number values must become string percentages.

Purpose: In react-resizable-panels v4, bare numbers passed to `defaultSize`, `minSize`, and `maxSize` on `<Panel>` are interpreted as pixels, not percentages. This causes the instructions panel to be capped at ~45px (the value of maxSize={45}), making it unusably narrow. The reset button also appears broken because clamping prevents the layout from changing. Fixing the props to string percentages (`'25%'`, `'15%'`, `'45%'`) unblocks three UAT failures (tests 1, 3, 4) with a single targeted change.

Output: src/App.tsx with all Panel size prop values changed to string percentages.
</objective>
